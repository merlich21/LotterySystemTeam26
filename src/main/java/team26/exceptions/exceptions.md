# Exception Handling

# Назначение

В приложении реализован единый механизм обработки ошибок для REST API.  
Он нужен для того, чтобы:

отделить бизнес-логику от HTTP-ответов;
возвращать клиенту ошибки в едином формате;
упростить сопровождение и расширение API;
избежать дублирования обработки ошибок в каждом обработчике маршрутов.

# Общая идея

В проекте используется два основных элемента:

1 кастомные исключения описывают тип ошибки;
2 глобальный error handler перехватывает исключения и преобразует их в HTTP-ответ.

Такой подход позволяет выбрасывать исключения в любом месте приложения, не формируя HTTP-ответ вручную в каждом контроллере или middleware.
